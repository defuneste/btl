Data products and more

A data product is a medium including data that is providing insights. The most common forms, in the wild, are reports, graphs, dashboards and web apps.

I want to add to that list “packaged code” for data or “data as code”.

Usually data is files storing information, the most commun example is probably tabular data in spreadhseet (.xls) or plain text files (.csv) but it can also be provided from a database either directly or exposed throught an API.

As writing API’s call can be tedious (even with amazing generic packages!) we have seen some great specific package implementations:

  • tidycensus1 on top of the US Census Bureau API2

  • nasapoawer3 on top of NASA POWER API client4

Those packages are already “data as code”: they allow you to seamlessly convert a lot operations in code and store them in plain text for version control and reproducibility. Sadly for reproducibility they are still depending on their upstream: if the US Census

After this lenghty introduction I want to add a new way: wrapping around duckdb, parquet and object storage. So far I have seen few use of that tech stack.
